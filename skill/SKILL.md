# 多智能体金融投研分析框架

## 核心设计理念

本框架受 **TradingAgents** 架构启发，模拟真实投资公司的多角色协作模式：

```
市场信息输入
    ↓
┌─────────────────────────────────────┐
│           分析师团队（并行）           │
│  基本面分析 | 情绪分析                │
│  行业分析  | 竞争分析                 │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│          多空辩论机制                 │
│     多头论点 vs 空头论点              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│           综合研报输出                │
│      投资评级 + 核心逻辑              │
└─────────────────────────────────────┘
```

## 交互模式

支持两种模式，**默认使用混合模式**：

### 模式 A：问答模式（快速响应）
当用户问题明确、只需特定角色分析时：
- 直接回答用户问题
- 自动识别需要哪个分析师角色
- 提供简洁的结论和建议

### 模式 B：深度流程模式（全面分析）
当用户请求深度分析（包含"分析"、"研究"、"评估"、"对比"等关键词），或用户明确要求时：
- 启动完整的多智能体协作流程
- 按顺序执行：分析师团队 → 多空辩论 → 综合研报

## 分析流程详解

### 第一步：理解用户需求

**必须明确以下信息：**
1. **标的类型**：股票（A股/港股/美股）、基金、行业指数
2. **标的名称或代码**：如 "贵州茅台 600519"、"中概互联 ETF"
3. **分析目的**：
   - 投资价值评估（买入/持有/卖出建议）
   - 风险识别（持仓检查、风险敞口）
   - 事件影响分析（财报、并购、政策影响）
   - 行业/板块比较（哪个更好）
   - 宏观传导分析（政策对行业的影响）

**如果信息不足**，先询问用户明确标的和分析目的，再开始分析。

### 第二步：分析师团队（并行分析）

调用以下分析师角色，**并行获取信息**：

| 分析师 | 职责 | 分析维度 |
|--------|------|----------|
| **基本面分析师** | 评估公司财务质量和内在价值 | 营收/利润/现金流、盈利能力、资产质量、估值水平 |
| **情绪分析师** | 评估市场情绪和预期 | 机构持仓、分析师评级、舆论情绪、资金流向 |
| **行业分析师** | 评估行业赛道和竞争格局 | 行业空间、竞争格局、产业链地位、政策环境 |
| **风险分析师** | 识别潜在风险因素 | 经营风险、财务风险、市场风险、宏观风险 |

**数据获取方式**：
- 使用 `finance-data-retrieval` 或 `neodata-financial-search` skill 获取行情、财务数据
- 使用 WebSearch 获取最新新闻、研报摘要
- 使用 `company-quality`、`dividend-returns` 等专业分析 skill 获取结构化评分

**输出格式**：每个分析师输出独立的分析段落，包含：
- 核心发现（3-5个关键点）
- 支撑数据（关键财务指标、市场数据）
- 简要结论（1-2句话）

### 第三步：多空辩论机制

基于分析师团队的发现，引导多空辩论：

**多头论点（Bull Case）**：
- 核心上涨逻辑（3-5个）
- 潜在催化剂（政策利好、业绩超预期、新产品/市场突破）
- 估值扩张的可能性

**空头论点（Bear Case）**：
- 核心下行风险（3-5个）
- 可能的负面情景
- 估值压缩的风险

**辩论结论**：
- 多空力量对比
- 当前市场定价了哪些预期
- 主要分歧点

### 第四步：综合研报输出

整合所有分析，输出结构化研报：

```markdown
# [标的名称] 投资研究报告

## 📊 标的概览
- 股票代码/ETF代码
- 所属行业/赛道
- 当前价格（如适用）
- 关键数据摘要

## 📈 基本面分析
### 核心财务指标
| 指标 | 数值 | 行业对比 |
|------|------|----------|
| ...  | ...  | ...      |

### 估值分析
- 当前估值（PE/PB/PS 等）
- 历史分位
- 与可比公司对比

## 🎭 情绪与预期
- 机构持仓变化
- 分析师一致预期
- 市场情绪指标

## 🏭 行业与竞争
- 行业空间
- 竞争格局
- 产业链位置

## ⚠️ 风险评估
- 上行风险
- 下行风险
- 主要风险因素

## 🐂🐻 多空辩论结论
### 多头逻辑
- ...

### 空头逻辑
- ...

### 核心分歧
- ...

## 💡 投资建议
### 综合评级
| 维度 | 评分/结论 |
|------|-----------|
| 基本面 | ⭐⭐⭐⭐☆ |
| 情绪面 | ⭐⭐⭐☆☆ |
| 行业前景 | ⭐⭐⭐⭐⭐ |
| 风险回报 | ⭐⭐⭐☆☆ |

### 操作建议
- **适合人群**：...
- **建议方式**：...
- **关键催化剂**：...
- **止损/止盈参考**：...

### 核心逻辑
（用 3-5 句话概括为什么是这个评级）

## 📅 后续跟踪
- 需要持续关注的数据节点
- 下次复盘时间
```

## LLM 模型配置

### 推荐配置

| 任务类型 | 推荐模型 | 说明 |
|----------|----------|------|
| 深度分析 | GPT-4o / Claude-3.5-Sonnet / DeepSeek-V3 | 需要强推理能力 |
| 快速问答 | GPT-4o-mini / Claude-3-Haiku / Qwen-Turbo | 成本更低，速度更快 |
| 中文市场 | 智谱 GLM-4 / 阿里 Qwen-Max / DeepSeek | 中文理解更好 |

### 模型切换

使用 `/model` 命令切换模型：
```
/model gpt-4o          # 切到 GPT-4o
/model deepseek-chat   # 切到 DeepSeek
```

## 注意事项

1. **免责声明**：分析结果仅供研究参考，不构成投资建议
2. **数据时效**：基于当前可获取的数据，定期更新
3. **模型限制**：LLM 存在幻觉风险，关键数据需二次验证
4. **风险教育**：建议用户理解风险后再做投资决策
